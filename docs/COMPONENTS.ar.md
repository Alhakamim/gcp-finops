## المكونات المشتركة

### مكونات واجهة المستخدم
- `Card` — غلاف مع تصميم متوافق مع الثيم
- `KPICard` — عرض المقاييس مع الرسوم الخطية
- `Badge` — مؤشرات الحالة
- `Change` — مؤشر النسبة المئوية (صعود/هبوط)
- `EnvBadge` — شارة البيئة
- `AlertDot` — نقطة خطورة التنبيه
- `SectionHeader` — عنوان القسم مع زر إجراء اختياري
- `SegmentControl` — تبديل مجموعة الأزرار

### الرسوم البيانية
- `BarChart` — رسم بياني شريطي SVG مع تدرجات
- `DonutChart` — رسم بياني دائري SVG
- `SparkLine` — رسم خطي مصغر SVG للاتجاهات

### التنقل
- `Sidebar` — مبدّل الحسابات + التنقل + الثيم واللغة
- `Header` — عنوان الصفحة + منتقي التاريخ + الإشعارات + قائمة المستخدم
- `CommandPalette` — Ctrl+K بحث سريع

## تدفق البيانات

```
تفاعل المستخدم
      ↓
تغيير حساب الفواتير → تحديث activeAccount?.id
تغيير نطاق التاريخ → تحديث dateStart/dateEnd
      ↓
useEffect [activeAccount?.id, dateStart, dateEnd]
      ↓
gcpApi.costData(id, dateStart, dateEnd) → طلب API
      ↓
الخادم يعيد بيانات الحساب من SQLite
      ↓
تحديث حالة المكون → إعادة العرض
```

## إدارة الحالة

التطبيق يستخدم `AppCtx` (React Context) مع:
- `dark` / `setDark` — حالة الثيم
- `lang` / `setLang` — حالة اللغة
- `t` / `T` — كائنات الترجمة والثيم
- `activeAccount` — حساب الفواتير الحالي
- `dateStart` / `dateEnd` — فلتر نطاق التاريخ
